ABSTRACTION
* Hiding implementation details from the user
* Achieved using abstract classes and interfaces
* Abstract class can have abstract and non-abstract methods
* Interface contains abstract methods by default (Java 8+ can have default and static methods)
* Allows multiple implementations and code flexibility

6
ENCAPSULATION
* Wrapping data (variables) and methods into a single unit (class)
* Hiding internal data using private access modifiers
* Providing public getter and setter methods to access and update data
* Helps in controlling how data is accessed or modified
* Increases code maintainability and security

7
INHERITANCE
* Mechanism where one class acquires properties of another class
* Superclass (parent) and subclass (child) relationship
* Reuse code and reduce redundancy
* Supports method overriding for runtime polymorphism
* Types: single, multilevel, hierarchical, multiple (via interfaces)
OPEN REPO
8
POLYMORPHISM
* Ability of an object to take many forms
* Compile-time (method overloading) and runtime (method overriding) polymorphism
* Improves code flexibility and reusability
* Achieved using inheritance and interfaces
* Example: Same method name performing different tasks based on parameters or object type
OPEN REPO
9
PACKAGES
* Grouping related classes and interfaces together
* Helps in avoiding name conflicts
* Provides access protection and modularity
* Types: built-in (java.util, java.io) and user-defined packages
* Use 'import' keyword to access classes from other packages
